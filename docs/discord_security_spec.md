# Discordサーバー設定「セキュリティ」メニュー運用仕様

## 1. 目的

本ドキュメントは、Discordサーバー設定内の「セキュリティ」メニュー（DM一時停止など）を、
運用者が誤解なく設定・監視できるようにするための運用仕様です。

本Botが自動更新する `dms_disabled_until` は、上記メニューのDM一時停止設定と対応します。

## 2. 対象範囲

- 対象画面: Discord のサーバー設定内「セキュリティ」メニュー
- 対象項目: DM一時停止（`dms_disabled_until`）
- 対象実装: `extensions/auto_disable.py` による自動更新

> 注意: Discordクライアント（PC/モバイル）やバージョンにより、文言・配置が異なる場合があります（未確認、必要に応じて確認）。

## 3. メニューの位置と確認手順

### 3.1 代表的な確認導線

1. 対象サーバーを開く
2. サーバー名メニューから「サーバー設定」を開く
3. 「セキュリティ」メニューを開く
4. DM一時停止の現在状態（有効期限）を確認

### 3.2 画面確認時の観点

- DM一時停止の有効期限が未来時刻になっているか
- 手動変更後にBot実行タイミングで再上書きされる設計であることを理解しているか
- 権限不足やAPI制約で更新できていないサーバーがないか

## 4. DM一時停止項目の仕様

### 4.1 機能概要

DM一時停止は、サーバー参加メンバー間のDMを一定期間制限するための設定です。
本Botはこの期限を定期更新し、常時保護に近い状態を維持します。

### 4.2 Botによる更新値

- 更新対象: `dms_disabled_until`
- 更新周期: 6時間ごと
- 更新値: 実行時点から24時間先

そのため、Botが正常稼働中はDM一時停止の期限が継続的に先送りされます。

### 4.3 手動設定との関係

- 管理者がセキュリティメニューで期限を短く変更しても、次回Bot実行で再延長されます。
- 管理者がDM一時停止を解除しても、次回Bot実行で再設定される可能性があります。
- 手動優先の運用に切り替える場合は、Bot停止または対象Guild除外実装が必要です。

## 5. 運用ポリシー

### 5.1 推奨ポリシー

- 新規流入が多いサーバーでは常時有効運用
- イベント時のみ緩和する場合は、緩和期間中にBotを停止する手順を事前定義
- 手動変更を行う担当者とBot運用担当者の責任分界を明確化

### 5.2 変更管理

- セキュリティメニュー設定を変更する場合は、変更理由・期間・戻し手順を記録
- Bot側の周期/延長時間を変更する場合はPRレビュー必須

## 6. 失敗時の挙動

### 6.1 権限不足時

- `guild.edit(dms_disabled_until=...)` で `discord.errors.Forbidden` が発生した場合、
  当該サーバーのみ更新失敗し、他サーバー処理は継続されます。

### 6.2 Bot停止時

- Bot停止中は期限更新が実行されません。
- 停止が長期化すると、DM一時停止の期限切れが発生する可能性があります。

## 7. 監視・点検項目

- 6時間ごとのタスク実行ログ有無
- `Forbidden` 発生サーバーの有無
- Discordセキュリティメニュー上での期限表示（抜き取り確認）
- Bot再起動後の更新再開確認

## 8. 本実装における既知事項

- セキュリティメニューのDM一時停止以外の項目（例: 招待制御、検証レベル等）は本Botの制御対象外です。
- 周期（6時間）と延長時間（24時間）はコード固定です。
- サーバー単位除外や通知連携は未実装です。

## 9. 参考（実装対応）

- 定期実行: `@tasks.loop(hours=6)`
- 更新処理: `guild.edit(dms_disabled_until=...)`
- 例外処理: `discord.errors.Forbidden` を捕捉して継続

実装詳細は `extensions/auto_disable.py` を参照してください。

## 10. コンフリクト（他設定がOFFになる問題）への対応

### 10.1 事象

セキュリティメニュー内のDM一時停止をBotが更新する際、
運用環境によっては他設定が意図せずOFFになるように見える、という報告があります（未確認、必要に応じて確認）。

### 10.2 技術的な整理

- 本Botは `guild.edit(invites_disabled_until=..., dms_disabled_until=...)` を実行し、
  DM更新時に招待一時停止値も同時反映します。
- それでも他設定に影響が出る場合、Discord側の仕様変更・UI反映差・同時編集競合が原因となる可能性があります。
- Bot単体でDiscord内部の全設定保存ロジックを完全制御することはできないため、
  「他項目へ絶対に影響しない」ことを保証する改修は不可能です。

### 10.3 実施した緩和策

- 招待一時停止の同時反映: `invites_disabled_until` を併せて送信

これにより、DM更新時に招待一時停止値が巻き戻る事象を回避します。

### 10.4 運用推奨

- 本番適用前に検証用Guildで、招待一時停止維持を確認
- セキュリティメニュー他項目の監査ログ/画面差分を確認後に本番展開
- 問題が継続する場合は当該Guildを対象外にし、手動運用へ切替


### 10.5 精査結果（招待一時停止以外）

`discord.py` の `Guild.edit` で incident actions として更新対象になるのは、
`invites_disabled_until` と `dms_disabled_until` です。
そのため、本Botの処理で同時反映が必要な同種項目は招待一時停止のみと判断しています。

なお、Discord側仕様変更時は再精査が必要です（未確認、必要に応じて確認）。
